Getting Started

Launch the Good Food restaurant app quickly and validate the full restaurant flow: login, orders, menu management, and reporting.

Goal: install dependencies, connect to your API, and complete one successful restaurant session.

1) Prerequisites

Before running the app, install Node.js and Expo tooling once for the suite, then make sure your backend API is reachable.

Important: Start your Good Food backend (for example the my-backend project) before opening the restaurant app. The mobile app only shows real data when the server is online and reachable from your phone or emulator.
Restaurant account: The email you use to log in must belong to a restaurant user on your backend. If you try a customer-only or driver-only account, access to restaurant screens may fail.

Install Node.js LTS, Expo tooling, and Android Studio / Xcode using the shared guide: Environment setup → Node.js and Expo and native tooling. When the app asks for location access, allow it if you want maps and GPS-based features.

2) Step-by-step installation

  1. Open a terminal.
  2. Go to the restaurant-app folder (the same level as package.json).
  3. Run npm install and wait until it finishes without red errors.
  4. Set API_BASE_URL in config.js.
  5. Confirm the backend is reachable from your test device.

When this completes, your computer has everything the project needs to start in development mode.

Install dependencies restaurant-app 
cd restaurant-app
npm install

3) Configuration & personalization

The restaurant app keeps its settings in one file at the root of the project: config.js. (The customer app uses config/index.js—different path, same idea.)

API address

The line that matters most is API_BASE_URL. It must match the address where your backend listens, usually ending with /api exactly as your server expects.

config.js (excerpt) 
API_BASE_URL: 'http://YOUR_SERVER_IP:5000/api',
APP_NAME: 'Good Food',
APP_SUBTITLE: 'Restaurant',
VERSION: '1.0.0',
DEMO_MODE: true,
DEMO_EMAIL: 'demo@restaurant.com',
DEMO_PASSWORD: 'password123',
API_TIMEOUT: 10000

Default development value is http://localhost:5000/api. On physical devices, replace localhost with your machine LAN IP.

Demo mode

When DEMO_MODE is true, the login screen can be pre-filled with sample credentials—handy for demos. Before a real release, set it to false and rely on real restaurant accounts.

Languages (EN/FR, lang/, i18n.js): see Environment setup → Languages (i18n).

Look and feel

Colors and shared styles are centralized in global.js. Adjust them there if you want the restaurant app to match your brand.

Tip: After editing config.js or theme files, stop and restart npm start so the bundler picks up changes.

Maps for production Android builds

Expo production build steps are shared across the suite. Follow Environment setup → Google Maps (Expo) if your build requires maps configuration.

4) API connection checklist

The app uses api.js for login, profile, restaurant stats, orders, order status updates, menu CRUD, and settings.

For a basic test you only need the correct URL in config.js. Device vs localhost is covered on Environment setup → API reachability.

5) Run the project

  1. Backend running, API_BASE_URL set, npm install already done.
  2. In restaurant-app, run npm start.
  3. Scan the QR code with Expo Go, or press a / i for Android or iOS simulator.
  4. Sign in with a restaurant account and validate Dashboard, Orders, Menu, Reports, and Settings.
Run commands Expo 
npm start

# optional shortcuts
npm run android
npm run ios
