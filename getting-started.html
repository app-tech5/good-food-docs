Getting Started

Launch the Good Food Pro customer app in minutes with a clear, beginner-friendly setup path.

Goal: complete installation quickly and run your first working app session.

1) Prerequisites

Before you can run the customer app, you need to install several tools. Don't worry, this guide will walk you through everything step by step, even if you're doing this for the first time.

Important: The customer app needs a running backend server. Make sure to start the my-backend server before launching this app.

Node.js Installation

Node.js is the foundation of this project. It allows you to run JavaScript on your computer and includes npm, the package manager we'll use to install all other tools.

We recommend using an LTS (Long Term Support) version for maximum stability. Choose your operating system below:

macOS
Windows
Linux
macOS - Using Homebrew (recommended) 
brew install node
Windows - Official Installer 
# 1. Go to https://nodejs.org
# 2. Download the LTS version
# 3. Run the installer (.msi file)
Linux - Using NodeSource 
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
Verify your installation (all platforms) 
node --version
npm --version

2) Step-by-Step Installation

  1. Open a terminal in the customer-app folder.
  2. Install dependencies with npm install.
  3. Wait for the installation to finish without errors.
  4. Confirm your backend API is up and reachable from your device/emulator.

This process gives you a ready local environment for development and demos.

Install Dependencies customer-app 
cd customer-app
npm install

3) Configuration & Personnalisation

The customer app is highly configurable. You can adapt it to your brand, choose your preferred language, enable demo mode for presentations, and customize the visual appearance.

API Configuration

The most important setting is the connection to your backend server. Edit the file config/index.js to set your API URL.

config/index.js 
API_BASE_URL: 'http://YOUR_SERVER_IP:5000/api',
APP_NAME: 'Good Food',
VERSION: '1.0.0'

Demo Mode

Perfect for demonstrations and client presentations. When enabled, the app automatically fills the login form with demo credentials.

Demo Configuration 
DEMO_MODE: true,
DEMO_EMAIL: 'demo@customer.com',
DEMO_PASSWORD: 'demo123'

Multi-language Support

The customer app is currently available in English and French. The system automatically detects the language of the user's device using expo-localization and displays the interface in the appropriate language.

To add a new language, you need to:

  1. Create a new translation file in the lang/ folder (for example es.json for Spanish)
  2. Copy the complete structure from en.json or fr.json and translate all the text
  3. Update the file lang/i18n.js to import and register your new language
  4. The app will then automatically support the new language for users who have their device set to that language

Visual Customization

You can customize colors, themes, and assets by editing the files in the global/ and config/assets.js folders.

Pro tip: After making configuration changes, restart the development server with npm start for the changes to take effect.

Maps Configuration for Production APK

The customer app displays restaurants on a map. When you test the app with npm start, the map works without any special setup.

However, when you create a real APK file to install on Android phones, Google requires an API key to display the maps correctly.

Step 1: Get a Google Maps API Key 
1. Open your browser and go to: https://console.cloud.google.com
2. Sign in with your Google account
3. Create a new project or select an existing one
4. In the search bar, type "Maps SDK for Android" and enable it
5. Go to "Credentials" and click "Create Credentials" → "API key"
Step 2: Add the key to your project 
{
  "expo": {
    "android": {
      "config": {
        "googleMaps": {
          "apiKey": "AIzaSyYourGoogleMapsAPIKeyHere123456789"
        }
      }
    }
  }
}

After adding your key, rebuild your APK with the command:
eas build --platform android or expo build:android

4) API Connection

The customer app sends requests to your backend API for authentication, restaurants, menus, cart actions, checkout, and order tracking updates.

Keep the API endpoint stable and reachable from the simulator/phone network. No code changes are needed here for this quick-start page.

Example: if your backend runs on 192.168.1.20:5000, your API base URL should use that same reachable IP instead of localhost when testing on a physical device.

5) Run the Project

  1. Start the app with npm start.
  2. Open on Android, iOS, or Expo Go from the generated QR code.
  3. Verify the core flow: sign in, browse restaurants, add to cart, checkout, track order.
Run Commands Expo 
npm start

# optional shortcuts
npm run android
npm run ios
npm run web
