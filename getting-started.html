Getting Started

Launch the Good Food customer app in minutes with a clear, beginner-friendly setup path.

Goal: complete installation quickly and run your first working app session.

1) Prerequisites

Before you can run the customer app, you need to install several tools. Don't worry, this guide will walk you through everything step by step, even if you're doing this for the first time.

Important: Start your backend (for example the my-backend project) before launching this app so login and API calls succeed.

Install Node.js LTS and the usual Expo / Android / iOS tooling once for the whole suite — the full walkthrough (macOS, Windows, Linux, verify commands) lives on Environment setup → Node.js.

2) Step-by-Step Installation

  1. Open a terminal in the customer-app folder.
  2. Install dependencies with npm install.
  3. Wait for the installation to finish without errors.
  4. Confirm your backend API is up and reachable from your device/emulator.

This process gives you a ready local environment for development and demos.

Install Dependencies customer-app 
cd customer-app
npm install

3) Configuration & Personnalisation

The customer app is highly configurable: branding, demo mode, and visuals. Languages (EN/FR and adding locales) are documented once for every app on Environment setup → Languages (i18n).

API Configuration

The most important setting is the connection to your backend server. Edit the file config/index.js to set your API URL.

config/index.js 
API_BASE_URL: 'http://YOUR_SERVER_IP:5000/api',
APP_NAME: 'Good Food',
VERSION: '1.1.0'

Demo Mode

Perfect for demonstrations and client presentations. When enabled, the app automatically fills the login form with demo credentials.

Demo Configuration 
DEMO_MODE: true,
DEMO_EMAIL: 'demo@customer.com',
DEMO_PASSWORD: 'demo123'

Visual Customization

You can customize colors, themes, and assets by editing the files in the global/ and config/assets.js folders.

Tip: After making configuration changes, restart the development server with npm start for the changes to take effect.

Maps Configuration for Production APK

Maps keys and app.json steps are shared by every Expo app in this suite. See Environment setup → Google Maps (Expo).

4) API Connection

The customer app sends requests to your backend API for authentication, restaurants, menus, cart actions, checkout, and order tracking updates.

Keep API_BASE_URL stable and reachable from the simulator or phone. Why localhost often fails on a real device is explained on Environment setup → API reachability.

5) Run the Project

  1. Start the app with npm start.
  2. Open on Android, iOS, or Expo Go from the generated QR code.
  3. Verify the core flow: sign in, browse restaurants, add to cart, checkout, track order.
Run Commands Expo 
npm start

# optional shortcuts
npm run android
npm run ios
npm run web
