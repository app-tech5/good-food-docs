Good Food — Customer App

Build a modern food-ordering experience your customers enjoy from day one.

This overview helps non-technical buyers and technical teams clearly understand what the customer app does, which platforms it supports, and what is needed to run it.

Easy to launch Android & iOS ready Self-hosted REST API

Overview (Presentation)

The Good Food customer app is the mobile storefront of your food delivery business. It turns restaurant browsing and ordering into a clear, smooth, and user-friendly journey.

This codebase is the updated Good Food stack: mobile clients talk to your own REST API and database. You host the backend (for example Express.js + MongoDB) and point the apps at your API base URL.

As the buyer-facing app in the suite, it is designed to take users from "I am hungry" to "Order confirmed" with minimal friction, while order and account data stay on your server.

Backend architecture

Data and auth go through your Good Food REST API. You run the API and MongoDB (or your chosen hosting), configure API_BASE_URL in the app, and scale or customize the backend like any Node.js service.

Customer App Presentation

This app is made for everyday users, not only advanced users. The interface is simple, visual, and action-oriented so even first-time customers can order with confidence.

Customers can sign in, discover nearby restaurants, explore menu items, add products to the cart, complete checkout, and follow order progress in a smooth, guided flow.

Beyond ordering, the app includes practical account features such as address management, language preferences, payment options, and order tracking visibility to improve retention and trust.

Main Function

The core mission is simple: let users place food orders from mobile, from item discovery to final confirmation, then keep them informed through order tracking.

In real usage, customers can search restaurants, filter by preferences, review menu details, customize quantities, and checkout in a few steps.

After checkout, customers can monitor order status updates from preparation to delivery, reducing uncertainty and improving satisfaction.

Supported Platforms

  • Android
  • iOS

A single shared codebase powers both platforms, which helps maintain consistent user experience and visual quality across devices.

This approach also lowers maintenance cost and speeds up feature delivery, updates, and bug fixes for product teams.

The customer app uses React Native and Expo for Android and iOS. State and navigation patterns are built for production ordering flows against your HTTP API.

Persistence and business rules live on your API and database; the mobile app consumes your HTTP endpoints.

Run the backend from the repository that ships with the suite (for example my-backend), then connect the app using config/index.js as described in Getting Started.

What you need to run the app

Node.js, Expo, simulators, API URLs, and Maps keys are documented once for the whole suite on Environment setup.

For this customer app specifically: clone customer-app, install dependencies, set API_BASE_URL in config/index.js, start the API from my-backend (see Backend — Getting Started), then follow Getting Started.
