Getting Started — Driver App

Install dependencies, point the app at your API, and run the Good Food Driver experience on a simulator or device.

You need a backend user with a driver profile. Plain customer accounts will fail the post-login driver check.

1) Prerequisites

Same baseline as the other Good Food mobile apps: Node.js LTS, npm, Expo tooling, and a reachable REST API.

Backend: Start your Good Food API (for example the my-backend project) before testing login and deliveries. Use a LAN IP or HTTPS tunnel URL that the phone or emulator can reach—not localhost from a physical device.
  • Verify node --version and npm --version.
  • Install Android Studio and/or Xcode if you use emulators or produce binaries.
  • Grant location permission when prompted; the app declares coarse/fine location on Android (app.json).

2) Install dependencies

  1. Open a terminal in the delivery-app folder.
  2. Run npm install.
  3. Wait until the install completes without errors.
Install delivery-app 
cd delivery-app
npm install

3) Configure config.js

All environment-specific settings for this app live in config.js at the project root (not under a config/ folder).

config.js (excerpt) 
API_BASE_URL: 'http://YOUR_SERVER_IP:PORT/api',
GOOGLE_MAPS_API_KEY: 'YOUR_GOOGLE_MAPS_API_KEY',
APP_NAME: 'Good Food Driver',
VERSION: '1.0.0',
DEMO_MODE: true,
DEMO_EMAIL: 'driver@demo.com',
DEMO_PASSWORD: 'driver123',
API_TIMEOUT: 10000

Set API_BASE_URL to the same API base you use for the customer app (including the /api suffix as implemented in your backend). Adjust port to match your server (common examples are documented in the file comments).

For production, set DEMO_MODE to false and remove reliance on demo credentials.

Driver login behavior

After /auth/login, the client calls /drivers/profile. If the user has no driver profile, login is rejected with an error—create or assign a driver profile on the server for test accounts.

4) Maps and location

Route and map features rely on react-native-maps and related libraries. Put your browser/SDK key in GOOGLE_MAPS_API_KEY where the app reads it.

For Android release builds, you may also need the Maps SDK key in Expo config (similar to the customer app): under expo.android.config.googleMaps.apiKey in app.json when you build with EAS or classic Expo builds.

5) Run the project

  1. Ensure the backend is running and API_BASE_URL is reachable.
  2. From delivery-app, run npm start.
  3. Open the app in Expo Go or an emulator; sign in with a driver account.
  4. Smoke-test: home screen loads, deliveries list or empty state, drawer navigation, logout.
Run Expo 
npm start

# optional
npm run android
npm run ios

6) Internationalization

Strings are driven by i18n.js and JSON files under lang/ (English and French). To add a language, copy an existing locale file, translate, and register it in the i18n setup the same way as in the customer app.
