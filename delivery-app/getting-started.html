Getting Started

Launch the Good Food driver app with the same gentle, step-by-step style as the customer app—even if this is your first mobile project.

Goal: install tools, connect to your API, and complete one successful login as a driver. You will need a user account that is registered as a courier on the server (not only a customer).

1) Prerequisites

Before the driver app can run, you need a few free tools on your computer. This section explains each one in plain language. Follow the steps for your operating system; you can always come back if something fails the first time.

Important: Start your Good Food backend (for example the my-backend project) before opening the driver app. The mobile app only shows real data when the server is online and reachable from your phone or emulator.
Driver account: The email you use to log in must belong to someone your backend recognizes as a driver. If you try a normal customer account, the app may refuse login after the password step. Create or promote a test driver in your database or admin tools first.

Install Node.js LTS, Expo tooling, and Android Studio / Xcode using the shared guide: Environment setup → Node.js and Expo and native tooling. When the app asks for location access, allow it if you want maps and GPS-based features.

2) Step-by-step installation

  1. Open a terminal.
  2. Go to the delivery-app folder (the same level as package.json).
  3. Run npm install and wait until it finishes without red errors.
  4. Confirm your backend is still running and that you know the API URL you will paste into config.

When this completes, your computer has everything the project needs to start in development mode.

Install dependencies delivery-app 
cd delivery-app
npm install

3) Configuration & personalization

The driver app keeps its settings in one file at the root of the project: config.js. (The customer app uses config/index.js—different path, same idea.)

API address

The line that matters most is API_BASE_URL. It must match the address where your backend listens, usually ending with /api exactly as your server expects.

config.js (excerpt) 
API_BASE_URL: 'http://YOUR_SERVER_IP:5000/api',
GOOGLE_MAPS_API_KEY: 'YOUR_GOOGLE_MAPS_API_KEY',
APP_NAME: 'Good Food Driver',
VERSION: '1.1.0',
DEMO_MODE: true,
DEMO_EMAIL: 'driver@demo.com',
DEMO_PASSWORD: 'driver123',
API_TIMEOUT: 10000

Use the same host and port you already use for the customer app. If the port in your backend is different (for example 8080), change the URL accordingly.

Demo mode

When DEMO_MODE is true, the login screen can be pre-filled with sample credentials—handy for demos. Before a real release, set it to false and rely on real courier accounts.

Languages (EN/FR, lang/, i18n.js): see Environment setup → Languages (i18n).

Look and feel

Colors and shared styles are centralized in global.js. Adjust them there if you want the driver app to match your brand.

Tip: After editing config.js or theme files, stop and restart npm start so the bundler picks up changes.

Maps for production Android builds

Console steps and app.json are the same for every Expo app here. Follow Environment setup → Google Maps (Expo), then keep GOOGLE_MAPS_API_KEY in config.js in sync if your code reads it from there.

4) API connection checklist

The driver app contacts your server for sign-in, order lists, profile updates, and settings—just like the customer app, but with driver-specific endpoints.

For a basic test you only need the correct URL in config.js. Device vs localhost is covered on Environment setup → API reachability.

5) Run the project

  1. Backend running, API_BASE_URL set, npm install already done.
  2. In delivery-app, run npm start.
  3. Scan the QR code with Expo Go, or press a / i for Android or iOS simulator.
  4. Sign in with a driver account; open the side menu and tap through Home, Deliveries, and Profile to confirm navigation works.
Run commands Expo 
npm start

# optional shortcuts
npm run android
npm run ios
