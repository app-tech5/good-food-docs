Good Food — Delivery (Driver) App

Mobile workspace for couriers: see assignments, update delivery status, track earnings, and stay on route.

This page summarizes what the driver app does in the Good Food suite, how it talks to your backend over REST, and how the codebase is organized.

Expo & React Native Drawer + stack navigation REST API + offline-friendly cache

Role in the ecosystem

The delivery app is the courier-facing companion to the customer and restaurant apps. Drivers sign in with an account that has a driver profile on the backend; the app then loads orders, status, and account data over HTTPS.

It uses the same Good Food REST API (Express + MongoDB) as the customer and restaurant apps. Authentication uses /auth/login, then the client verifies a driver profile with /drivers/profile.

Customer app documentation

What drivers can do

  • Sign in and maintain session (token stored via AsyncStorage).
  • View a home dashboard and toggle availability / delivery-related status (via driver status API).
  • Browse and open active deliveries; update order progress along the delivery workflow.
  • See earnings, transactions, delivery history, and summary reports.
  • Manage profile and vehicle details; adjust app settings and theme-related preferences.
  • Use support and notifications screens; optional flows include ratings and payment methods when enabled in navigation.

Maps and location (expo-location, react-native-maps) support routing and on-duty behavior—ensure location permissions and map keys are configured for real devices and store builds.

Main function

Connect authenticated drivers to live order data, let them accept and progress deliveries, and surface earnings and history in one place.

The DriverProvider composes authentication, driver statistics, and order/delivery hooks so screens share one source of truth and can invalidate caches after actions.

Navigation model

Root stack (navigation/AppNavigator.js): Splash → Login → Drawer when authenticated. Navigation resets when auth state changes.

Drawer (navigation/DrawerNavigator.js): Home, Deliveries (stack), Earnings, Transactions, History, Reports, Notifications, Support, Profile (stack), Settings (stack). Some routes (e.g. order details) are hidden from the drawer but reachable from other screens.

API layer (summary)

A single client in api.js wraps fetch with JSON headers, bearer token, and timeout. It implements driver login (including post-login check for a driver profile), registration, status updates, orders, settings, profile updates, payment methods, and related resources.

Demo mode (config.js): when DEMO_MODE is true, parts of the stack can use mock or simplified data paths for demos—disable for production and point API_BASE_URL at your deployed API.

Supported platforms

  • Android (com.goodfood.driver)
  • iOS (com.goodfood.driver)

Built with Expo; one JavaScript codebase targets both platforms. Web script exists in package.json but the product is intended for native mobile.
