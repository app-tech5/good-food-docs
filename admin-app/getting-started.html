Getting Started

Bring the Good Food admin dashboard online with the same beginner-friendly recipe as the customer app: prerequisites, install, configure, run.

Goal: install dependencies, point the UI at your API, sign in with an administrator account, and open at least one real data screen.

1) Prerequisites

You need Node.js on your computer and a running Good Food backend. The admin interface is a standard React dev server in local development; in production you would build static files and host them behind HTTPS like any internal tool.

Important: Start your backend (for example my-backend) before npm start on the admin project. The dashboard immediately requests protected data after login; an offline API produces errors that look scarier than they are.
Admin account: Use credentials your server recognises as privileged (admin, manager, or whatever your backend encodes). Limited roles might sign in but see empty sections—this is usually a permissions issue, not a broken install.

Install Node.js LTS: Environment setup → Node.js. Languages (EN/FR) for all apps: Environment setup → Languages (i18n).

2) Step-by-Step Installation

  1. Open a terminal.
  2. Enter the admin-app folder (next to package.json).
  3. Run npm install and wait for a clean finish.
  4. Open the configuration section below—do not skip the API URL step.

This downloads the React tooling the repository expects. No database runs inside the admin repo itself; your API remains the source of truth.

Install dependencies admin-app 
cd admin-app
npm install

3) Configuration & Personalization

The admin project keeps its networking defaults in a small config folder inside src. You do not need to memorise file names: follow the example block, paste your real URL, save, restart the dev server if needed.

API configuration

Set the API base URL to the same origin your mobile apps use—usually ending with /api. Local development often starts with http://localhost:5000/api on the same machine as the backend.

Example API and preset credentials (adjust to match your backend) 
API_BASE_URL = 'http://YOUR_SERVER_IP:5000/api'

// optional: quick-fill presets used on the login screen in dev
roleCredentials = {
  admin: { email: 'admin@example.com', password: 'admin123' },
  manager: { email: 'manager@example.com', password: 'manager123' }
}

Replace sample emails and passwords with users that actually exist in your database. Troubleshooting reachability: Environment setup → API reachability.

Interface language

Translation bundles live with the admin source; follow the suite-wide i18n guide linked above if you need an extra locale or wording tweak.

Look and feel

Global styling is applied from the project’s main CSS entry; individual screens and widgets may add their own scoped styles. For brand tweaks, cooperate with your designer rather than editing random components without a plan.

Tip: After changing API defaults, restart npm start so every module reads the fresh values.

4) API Connection

Everything meaningful—session check, listing restaurants, editing a driver profile, exporting a report—flows through HTTP calls to your backend. The admin repo already wires those calls; your responsibility is a reachable URL and valid credentials.

If lists stay empty, verify the user’s role on the server and that pagination filters are not hiding data before assuming the UI is broken.

5) Run the Project

  1. Backend up; API base URL saved; npm install done.
  2. From admin-app, run npm start.
  3. Your browser should open http://localhost:3000 (or follow the terminal link).
  4. Log in with an admin-capable user and click through dashboard, one list screen, and settings.
Run commands React Scripts 
npm start
