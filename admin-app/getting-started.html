Getting Started

Launch the Good Food admin dashboard quickly and validate the core admin flow: login, dashboard, entity management, and settings.

Goal: install dependencies, connect to your API, and complete one successful admin session.

1) Prerequisites

Before running the app, install Node.js once for the suite, then make sure your backend API is reachable.

Important: Start your Good Food backend (for example the my-backend project) before opening the admin app. The dashboard loads protected resources and requires a live API.
Admin account: Use credentials with admin/manager permissions. Limited roles may authenticate but not have access to all dashboard sections.

Install Node.js LTS using the shared guide: Environment setup → Node.js . Language setup is shared as well: Environment setup → Languages (i18n).

2) Step-by-step installation

  1. Open a terminal.
  2. Go to the admin-app folder (the same level as package.json).
  3. Run npm install and wait until it finishes without red errors.
  4. Set API_BASE_URL in src/config/api.js.
  5. Confirm the backend is reachable from your test device.

When this completes, your computer has everything the project needs to start in development mode.

Install dependencies admin-app 
cd admin-app
npm install

3) Configuration & personalization

The admin app stores API and auth defaults in src/config/api.js and src/config/config.js. App settings (branding/runtime options) are also loaded from backend settings endpoints.

API address

The line that matters most is API_BASE_URL. It must match the address where your backend listens, usually ending with /api exactly as your server expects.

src/config/api.js + src/config/config.js (excerpt) 
API_BASE_URL = 'http://YOUR_SERVER_IP:5000/api'

// role presets used by login quick-fill
roleCredentials = {
  admin: { email: 'admin@example.com', password: 'admin123' },
  manager: { email: 'manager@example.com', password: 'manager123' }
}

Default development value is http://localhost:5000/api. If your backend runs elsewhere, update it before login tests.

Demo mode

The login screen provides role buttons (admin/manager/client) to prefill credentials from config. Keep them aligned with users that exist in your backend environment.

Languages (EN/FR, src/locales/, src/i18n.js): see Environment setup → Languages (i18n).

Look and feel

Global styles are in src/index.css and feature styles in component/page SCSS files. Theme consistency can also be managed from runtime settings APIs.

Tip: After editing API or config files, restart npm start to avoid stale environment state.

4) API connection checklist

The app uses src/api/apiClient.js and related services for login checks, entity CRUD, settings, languages, currencies, and reporting endpoints.

For a basic test you only need the correct URL in src/config/api.js. API network details are covered on Environment setup → API reachability.

5) Run the project

  1. Backend running, API_BASE_URL set, npm install already done.
  2. In admin-app, run npm start.
  3. Open http://localhost:3000 in your browser if it does not open automatically.
  4. Sign in with an admin account and validate Dashboard, main entities, Reports, and Settings.
