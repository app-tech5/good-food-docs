Good Food — Admin App

The central back-office web dashboard for the Good Food suite: manage users, restaurants, drivers, orders, finance, settings, and reports from one place.

This overview explains how the admin panel is structured in code, how dynamic routes and entities work, and which backend capabilities are required for production operations.

Easy to launch Web dashboard ready Connects to your Good Food API

Overview (Presentation)

The admin app is the control center of Good Food. It is a React web application used by admins/managers to manage core resources and global settings across customer, restaurant, and driver products.

It connects to the same REST API as mobile apps (/api) and authenticates via cookie/session checks (for example /users/me) before loading private routes.

Customer app — Overview Restaurant app — Overview Driver app — Overview

Main function

Centralize administration tasks: CRUD on business entities, financial monitoring, settings management, and operational reporting in one web interface.

The app generates most routes from the entities registry and uses reusable list/form pages to avoid duplicating CRUD screens for each resource.

How navigation is organized

Entry point is src/App.js with React Router. PrivateRoute guards access: authenticated users see Home, unauthenticated users are redirected to Login.

Sidebar items are built dynamically from src/data/entities.js. Routes are generated by src/routes/generateRoutes.js for list/detail/new pages.

Configuration & demo mode

Core admin settings are split between src/config/api.js (API base URL), src/config/config.js (defaults/credentials), and app-level settings endpoints.

The login screen includes quick role presets (admin/manager/client) for testing, while production auth relies on backend credentials and secured cookies.
