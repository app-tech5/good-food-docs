Getting Started

Run the Good Food API locally: install dependencies, connect MongoDB, apply migrations, and confirm the server answers on port 5000.

Goal: a healthy GET or login flow against /api before you open any mobile or web client.

1) Prerequisites

You need a terminal, Node.js LTS, and a reachable MongoDB. The commands below are copy-paste friendly; full OS-specific Node install steps are on Environment setup → Node.js.

What this stack does

my-backend is the JSON API (port 5000 by default). Node.js runs it; npm installs dependencies from package.json. MongoDB stores data—if it is down, fix Mongo before npm start.

Order: Mongo running → .env → migrations → start the server.

Open the project folder

The my-backend project lives on your machine like any other folder (for example next to customer-app). Open a terminal there—you should see package.json.

Shell
bash / zsh / PowerShell 
cd path/to/my-backend

Verify Node.js and npm

Use a new terminal after installing. Expect something like v20.x.x and 10.x.x. “Command not found” means Node is missing from PATH.

Versions
Node LTS 
node -v
npm -v

MongoDB (via Docker)

This backend uses MongoDB running in Docker.

Install + run MongoDB
docker 
docker pull mongo:7
docker run -d --name <mongo_container_name> -p <mongo_host_port>:27017 mongo:7
docker ps
.env
required 
MONGO_URI=mongodb://127.0.0.1:<mongo_host_port>/<mongo_database_name>

Ready?

Checklist
before step 2 
# You should have:
# - node -v and npm -v working
# - Mongo reachable at the URI you will put in .env
# - a plain-text editor for .env (no smart quotes)
Secrets: Never commit MONGO_URI or JWT_SECRET. Keep them in .env only and do not publish that file.

Phones and emulators: localhost is not your laptop—see API reachability.

2) Step-by-Step Installation

  1. Place the my-backend folder next to your other Good Food apps (or wherever you keep the suite).
  2. Open a terminal in the my-backend folder (where package.json lives).
  3. Run npm install.
  4. Create a .env file in the project root (see next section).
Install dependencies
my-backend 
cd my-backend
npm install

3) Environment variables

The server reads configuration from .env via dotenv. In your backend code, src/config/db.js requires MONGO_URI explicitly, so keep this value valid first. Then set JWT_SECRET, PORT, and CORS_ORIGINS for your client URLs.

.env (example)
env 
MONGO_URI=mongodb://127.0.0.1:<mongo_host_port>/<mongo_database_name>
JWT_SECRET=<your_super_secure_jwt_secret>
PORT=<api_port_ex_5000>
NODE_ENV=development
CORS_ORIGINS=<admin_origin_1>,<admin_origin_2>

Match PORT and /api to what you put in customer, restaurant, driver, and admin configs (for example http://localhost:5000/api).

Backend project name & port

The backend doesn't display a “brand name” in the UI like Expo apps do. However, the backend project itself and its runtime host/port are configured here.

1) Rename the backend project (optional)

If you rename the folder or want a different tooling name, update my-backend/package.json.

package.json (project name)
my-backend 
{
  "name": "your-backend-project-name",
  "version": "1.0.0"
}

2) Change the server port (important)

If you change PORT, update every client’s API_BASE_URL so mobile apps and the admin dashboard still reach your server's /api.

.env (PORT)
env 
PORT=5000
CORS_ORIGINS=http://localhost:3000,http://localhost:19006

After changes, restart the backend with npm start.

4) Database & migrations

Migrations create collections and seed baseline data (languages, settings, demo users, etc.). Run them once on a fresh database before relying on the apps.

Database commands
npm 
npm run test:db
npm run migrate:up
npm run migrate:status

Full migration order and rollback notes are in README.md inside the my-backend repository.

5) Run the server

  1. MongoDB running; .env present; migrations applied.
  2. Start the process: npm start (or npm run dev if you have configured nodemon—see package.json in the repo).
  3. Confirm the console logs the listening port (default 5000).
  4. From a client app, set API_BASE_URL to http://<host>:5000/api and test sign-in.
Start API
server 
npm start

# or during development
npm run dev

6) Verify & next steps

Use the bundled Postman collection (postman_collection.json) or your client apps to hit auth and a simple resource route. When the API responds correctly, proceed to Customer Getting Started or any other app guide—always with the backend still running.
