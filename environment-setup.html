Environment setup

Node.js, Expo tooling, how to reach your API from a phone, and Google Maps keys for production builds — one place for the whole suite, so we do not repeat the same steps on every page.

Then continue with Customer app — Getting Started or Driver app — Getting Started.

Node.js

Node.js runs JavaScript on your computer and ships with npm, which installs the mobile apps' dependencies. Use an LTS (Long Term Support) release (for example 18.x or 20.x).

Install for your operating system:

macOS
Windows
Linux
macOS — Homebrew (recommended) 
brew install node
Windows — official installer 
# 1. Go to https://nodejs.org
# 2. Download the LTS version
# 3. Run the installer (.msi file)
Linux — NodeSource (Debian/Ubuntu example) 
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
Verify (all platforms) 
node --version
npm --version

Expo and native tooling

Every Good Food mobile app in this documentation uses React Native with Expo. You will typically need:

  • Expo CLI / Expo tooling (follow the official Expo docs for the current install command, or use npx expo via your project's scripts)
  • Android Studio for Android emulators and store builds
  • Xcode on macOS for the iOS simulator and App Store builds
  • A running Good Food REST API (for example the my-backend project) reachable from your machine and from any physical device you test with

Keep Node and the native tools on current stable releases to avoid odd build failures.

API reachability (devices vs localhost)

In your app config, API_BASE_URL must point to a host and port your phone or emulator can actually reach.

Physical device: If the backend runs on your laptop at 192.168.1.20:5000, use that LAN IP in the URL — not localhost, because on the phone "localhost" means the phone itself, not your computer.

Simulators on the same machine can sometimes use special host aliases (for example Android emulator's 10.0.2.2); use whatever your platform documents for reaching the host from the guest.

Google Maps (Expo, production Android)

During development with npm start, maps often work without extra setup. For a real APK/AAB, Google usually expects a Maps API key in your Expo Android config. The steps are the same for the customer and driver apps.

Step 1 — Google Cloud Console 
1. Open https://console.cloud.google.com
2. Create or select a project
3. Enable "Maps SDK for Android" (and Maps SDK for iOS if you ship to the App Store)
4. Credentials → Create credentials → API key
Step 2 — app.json (Expo) 
{
  "expo": {
    "android": {
      "config": {
        "googleMaps": {
          "apiKey": "YOUR_KEY_HERE"
        }
      }
    }
  }
}

If your project also reads a key from config/index.js (customer) or config.js (driver), keep that value in sync. Rebuild with eas build or your usual Expo workflow after changing keys.
